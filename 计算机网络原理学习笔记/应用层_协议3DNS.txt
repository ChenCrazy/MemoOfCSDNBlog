DNS：域名系统
因特网主机，路由器，都是使用IP地址(32bit)，用于数据报寻址
当使用域名系统后，使人易于记忆

IP地址和域名之间如何映射转换？就需要DNS来做工作

DNS：Domain Name System:
    分布式数据库：有许多域名服务器按层次构成
    应用层协议：主机、路由器、域名服务器互相通信进行域名解析
    注意：因特网的核心功能，应用层协议。网络边缘上的复杂实体
实际上就是一张表，分布在很多服务器上

为什么不搞集中的DNS?
 单点失败的问题
 数据的流通量有限
 远程集中式的数据库  维护问题
没有服务器能够保存所有Name-to-Ip地址的映射
 本地域名服务器：
 每个ISP，企业可拥有本地(默认)域名服务器
 主机的DNS查询首先在本机DNS管理文件中查找，然后发往本地域名服务器查询IP，若没有再由服务器之间交互查找，当然也是有时限的
授权域名服务器
每台主机必须在授权服务器上注册登记
可完成域名/地址的转换

世界各地遍布看13个根域名服务器:
当本地域名服务器不能解析时，就向根域名服务器查询
如果域名映射未知，则向授权夜明服务器查询，取得映射将映射返回本地域名服务器

递归查询：
一层一层向上查询，一级一级的返回过来，对根域名服务器造成工作负担
迭代查询：
被查询的服务器直接把可查询的服务器地址报回，减少中间环节

缓存和更新记录
一旦任何域名服务器得知了某个映射，就将其缓存，在一定时间间隔后缓存的条目将会过期(自动消除)

DNS记录
DNS：由世界上所有服务器所存储资源记录(RR)而组成的分布式数据库
    RR格式：(name,value,type,ttl )
    Type = A
        name=主机名 value=IP地址
    Type = NS
        name=域 value=该域授权域名服务器的IP地址
    Type = CNAME
        name=别名 value=真名
    Type = MX
        name=与name相关的邮件服务器域名

DNS协议，报文
DNS协议：查询和应答报文，二者格式相同，有一个标记为，称为flags
    报文首部
    indentification:16bit 用于查询，应答报文使用同样的
    flags: 查询或应答，希望递归还是可以递归，授权应答，，所有的报文都是一样的就需要flags来说明是查询还是应答报文
    Name,type fields，查询报文
    RRS响应
    来自授权服务器的记录
